Welcome to the tldraw blog! This is a sample post created to test the markdown content pipeline.

## What's new

We're moving from a CMS-based content system to a local, markdown-first pipeline. Content lives in the repo as `.md` files and gets loaded into a SQLite database at build time.

## Why markdown?

Markdown is simple, portable, and version-controlled. No external service dependencies, no API keys to manage, and every change is tracked in git.
