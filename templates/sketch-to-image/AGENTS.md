# AGENTS.md — sketch-to-image template

Guidance for AI coding agents working in this template. The repo-root `AGENTS.md` (monorepo conventions, sentence case, `yarn` usage, etc.) still applies — this file covers only what is specific to this template.

## What this is

A realtime sketch-to-image generator built on the tldraw SDK and powered by [fal.ai](https://fal.ai), with prompts auto-written from the sketch by [Claude](https://www.anthropic.com). The user draws on a tldraw canvas; each edit is rasterized, described into an image prompt by Claude vision, and sent to an LCM image-to-image model, with the generated image updating live in a side panel. A secondary "Animate → video" action turns the current image into a short clip.

It ships as a standalone starter template (a Vite app + a Cloudflare Worker), not as part of the SDK. Keep it self-contained and approachable — it is meant to be read and forked, so favor clarity over cleverness.

## Commands

Run from the repo root (not this directory):

- `yarn dev-template sketch-to-image` — run this template locally.
- `yarn typecheck` — type check. Never run bare `tsc`.

`package.json` sets `lint` to `null` in `tldraw_template`, so this template opts out of repo linting.

## Environment

Generation requires a fal.ai key and prompt auto-writing requires an Anthropic key. Create `.dev.vars` in this directory with `FAL_KEY=...` (fal.ai/dashboard/keys) and `ANTHROPIC_API_KEY=...` (console.anthropic.com). Both are server-side only — the worker injects them and they never reach the browser. For deploys: `wrangler secret put FAL_KEY` and `wrangler secret put ANTHROPIC_API_KEY`. Without the fal key, generation fails with a clear panel error; without the Anthropic key, auto-prompt fails but the user can still type a prompt.

## Architecture

Two halves: a Vite React frontend (`src/`) and a Cloudflare Worker (`worker/`).

**Realtime loop** (`src/realtime/`):

- [useRealtimeGeneration.ts](src/realtime/useRealtimeGeneration.ts) — the hook that drives everything. Listens to `editor.store` for user document changes, debounces them until the stroke settles, captures a frame, and — when the prompt is on auto — describes it before generating. Controls, the pause flag, and the auto/pinned flag are mirrored into refs so the stable `sendFrame` reads current values without re-subscribing on every keystroke — preserve this pattern when editing.
- [captureSketch.ts](src/realtime/captureSketch.ts) — rasterizes shapes to a fixed 512×512 white square via `editor.toImage` + letterboxing. The fixed square is deliberate: it keeps the sketch→image mapping stable frame to frame instead of jumping as bounds change.
- [describeSketch.ts](src/realtime/describeSketch.ts) — posts the captured sketch to the `/api/describe` worker route (Claude vision) and returns a short image prompt. Abortable, so a newer settled frame supersedes an in-flight describe.
- [falConnection.ts](src/realtime/falConnection.ts) — posts each frame to fal's synchronous HTTP endpoint through the worker proxy. Uses a monotonic request id + `AbortController` so a slow earlier response can never overwrite a newer frame. Despite the `RealtimeConnection` name, this is HTTP, not a WebSocket (chosen to avoid realtime token-scoping fragility).

**Worker** (`worker/`):

- [worker.ts](worker/worker.ts) — itty-router with three routes: `/api/fal/proxy`, `/api/describe`, and `/api/animate`.
- [routes/falProxy.ts](worker/routes/falProxy.ts) — validates the `x-fal-target-url` header points at an allowed fal host, injects `Authorization: Key <FAL_KEY>`, and relays the response. All realtime traffic flows through here.
- [routes/describe.ts](worker/routes/describe.ts) — sends the sketch to Claude vision with `ANTHROPIC_API_KEY` injected server-side and returns a one-line prompt. This is what lets the user skip writing a prompt.
- [routes/animate.ts](worker/routes/animate.ts) — the non-realtime image-to-video action (Seedance). Intentionally minimal: submits and waits synchronously. Production would move to fal's async queue with polling.

**Config** ([constants.ts](src/constants.ts)) — the model id, capture size, debounce (also the sketch-settle delay), and default controls live here as the primary customization surface.

## Conventions specific to this template

- New realtime fal traffic must go through the `/api/fal/proxy` worker route, and Claude traffic through `/api/describe`, so `FAL_KEY` and `ANTHROPIC_API_KEY` stay server-side — never call fal or Anthropic directly from `src/`.
- Keep the realtime path resilient to overlapping requests (supersede in-flight work); don't render stale frames or let a stale describe overwrite a typed prompt.
- When adding generation parameters, thread them through `GenerationControls`, `DEFAULTS`, and `RealtimeInput` together.
- `worker-configuration.d.ts` is generated by wrangler — do not hand-edit it beyond keeping the `Env` bindings in sync when adding a secret.
