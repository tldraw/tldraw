# tldraw Mermaid Starter Kit

A two-way converter between Mermaid diagram code and tldraw shapes. This starter kit demonstrates how to build custom shapes, tools, and integrations with external services like Google Gemini AI.

## Features

- **Code block shape** with syntax highlighting using Shiki
- **Code → Shapes conversion** for supported Mermaid diagram types
- **Shapes → Code conversion** using Google Gemini 2.5 Flash vision API
- **Manual conversion** triggered by user actions (not automatic sync)
- **Comprehensive logging** for debugging conversions
- **Native shape support** for flowcharts, sequence diagrams, class diagrams, state diagrams, and ER diagrams
- **SVG fallback** for unsupported diagram types

## Getting started

### Prerequisites

- Node.js 18+ and yarn
- Google Gemini API key (get one at https://aistudio.google.com/app/apikey)

### Installation

1. Install dependencies:

```bash
yarn install
```

2. Create a `.env` file with your Gemini API key:

```bash
cp .env.example .env
# Edit .env and add your API key
```

3. Start the development server:

```bash
yarn dev
```

## Usage

### Create a code block

1. Click the "Code Block" button in the toolbar (or press `C`)
2. Draw a rectangle on the canvas
3. Double-click to edit the code
4. Click "Convert to Diagram" to generate shapes from the code

### Convert shapes to code

1. Select shapes on the canvas
2. Click "Convert to Code" in the toolbar (or press `Shift+M`)
3. A code block will be created with the Mermaid code generated by Gemini

## Supported diagram types

### Native shape conversion

These diagram types are converted to native tldraw shapes (rectangles, diamonds, arrows, etc.):

- **Flowcharts** (`flowchart LR`, `flowchart TB`, etc.)
- **Sequence diagrams** (`sequenceDiagram`)
- **Class diagrams** (`classDiagram`)
- **State diagrams** (`stateDiagram`)
- **ER diagrams** (`erDiagram`)

### SVG fallback

Other diagram types are rendered as SVG images:

- Gantt charts
- Pie charts
- User journey diagrams
- Git graphs
- Mind maps
- Timeline diagrams
- Quadrant charts
- Requirement diagrams
- C4 diagrams

## Architecture

### Custom shapes

**CodeBlockShape** (`src/shapes/CodeBlockShape.ts`)

- Defines the shape type with props: `code`, `language`, `w`, `h`
- Uses module augmentation to register with tldraw

**CodeBlockShapeUtil** (`src/shapes/CodeBlockShapeUtil.tsx`)

- Renders the code block with syntax highlighting
- Handles editing mode with textarea
- Displays "Convert to Diagram" button when not editing

### Custom tools

**CodeBlockTool** (`src/tools/CodeBlockTool.ts`)

- Extends `BaseBoxShapeTool` for creating code block shapes
- Allows click-and-drag to create code blocks

### Conversion utilities

**convertCodeToShapes** (`src/utils/convertCodeToShapes.ts`)

- Detects Mermaid diagram type
- Routes to appropriate parser (flowchart, sequence, class, state, ER)
- Creates native tldraw shapes or falls back to SVG rendering
- Uses comprehensive logging for debugging

**convertShapesToCode** (`src/utils/convertShapesToCode.ts`)

- Exports selected shapes as PNG image
- Sends to Google Gemini 2.5 Flash vision API
- Receives Mermaid code in response
- Uses carefully crafted prompt for accurate diagram recognition

### Parsing utilities

Located in `src/utils/`:

- `parseMermaidFlowchart.ts` - Parse flowchart syntax
- `parseSequenceDiagram.ts` - Parse sequence diagram syntax
- `parseClassDiagram.ts` - Parse class diagram syntax
- `parseStateDiagram.ts` - Parse state diagram syntax
- `parseErDiagram.ts` - Parse ER diagram syntax
- `renderMermaidToSvg.ts` - Render any Mermaid code to SVG

### Shape creation utilities

Located in `src/utils/`:

- `createShapesFromFlowchart.ts` - Create shapes from parsed flowchart
- `createShapesFromSequenceDiagram.ts` - Create shapes from parsed sequence diagram
- `createShapesFromClassDiagram.ts` - Create shapes from parsed class diagram
- `createShapesFromStateDiagram.ts` - Create shapes from parsed state diagram
- `createShapesFromErDiagram.ts` - Create shapes from parsed ER diagram

### Detection utilities

**mermaidDetection** (`src/utils/mermaidDetection.ts`)

- `isMermaidCode()` - Check if text is Mermaid code
- `extractMermaidCode()` - Extract Mermaid code from various formats
- `getDiagramType()` - Identify the diagram type

### Logging

**logger** (`src/utils/logger.ts`)

- Comprehensive logging for all conversion operations
- Logs parsing, shape creation, and API calls
- Grouped console output for easy debugging

## Customization

### Add support for new diagram types

1. Create a parser in `src/utils/parse[DiagramType].ts`
2. Create a shape creator in `src/utils/createShapesFrom[DiagramType].ts`
3. Update `convertCodeToShapes.ts` to route to your new parser
4. Update `mermaidDetection.ts` to recognize the new diagram type

### Customize syntax highlighting

Edit `CodeBlockShapeUtil.tsx` and change the Shiki theme:

```tsx
codeToHtml(shape.props.code, {
  lang: shape.props.language,
  theme: 'your-theme-name', // e.g., 'github-light', 'monokai'
})
```

### Customize Gemini prompts

Edit `convertShapesToCode.ts` and modify the prompt to improve diagram recognition or support different output formats.

## Troubleshooting

### "Gemini API key not configured"

Make sure you have:
1. Created a `.env` file in the template directory
2. Added `VITE_GEMINI_API_KEY=your_api_key_here`
3. Restarted the dev server

### Shapes not converting correctly

Check the browser console for detailed logs. The logger outputs:
- Detected diagram type
- Parsed structure
- Shape creation operations
- API requests and responses

### Syntax highlighting not working

Shiki downloads themes and language grammars on first use. Check:
1. Network connection is available
2. No CORS errors in console
3. The language name is correct (use 'mermaid' for Mermaid code)
